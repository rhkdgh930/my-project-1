# Prompt Template

이 문서는 Codex에게 작업을 요청할 때 사용할 수 있는 짧은 템플릿이다. 정책 문서는 아니며, 세부 정책은 각 `docs/ai/*.md`를 따른다.

## 구현 요청 템플릿

목표:
- 무엇을 바꾸려는지 한 문장으로 적는다.

먼저 읽을 문서:
- `CODEX.md`
- 관련 `docs/ai/*.md`

현재 정책:
- 기존 동작과 제약을 적는다.

원하는 정책:
- 새 동작과 호환성 조건을 적는다.

주의:
- 건드리면 안 되는 도메인, API shape, unrelated cleanup 금지를 적는다.

테스트:
- 필수 시나리오와 실행 명령을 적는다.

작업 후 보고:
- 변경 요약, 테스트 결과, 남은 TODO를 요청한다.

## 리뷰 요청 템플릿

목표:
- 코드 수정 없이 구조와 방향성을 평가해 달라고 명시한다.

검토 기준:
- 정합성
- 성능
- 장애 복구
- 구현 복잡도
- 운영 난이도
- 테스트 신뢰성
- 포트폴리오 설득력

응답 형식:
- 결론을 먼저 요구한다.
- 장점, 단점, 위험, 추천 로드맵을 나눠 달라고 요청한다.

## 문서 수정 요청 템플릿

목표:
- 어떤 정책 변경을 어느 문서에 반영할지 적는다.

수정 범위:
- 문서만 수정
- 코드/테스트 수정 금지

반영할 내용:
- 추가/수정/삭제할 정책을 bullet로 적는다.

작업 후 보고:
- 수정한 문서
- 반영한 정책
- 코드/테스트 수정 여부

## 주의사항

- 긴 예시 프롬프트를 이 문서에 계속 누적하지 않는다.
- 현재 프로젝트 정책과 충돌하는 예시는 추가하지 않는다.
- 실제 작업 기준은 `docs/ai/testing.md`와 각 도메인 문서를 우선한다.
